# Partner Targets

## Integration Roadmap

### Phase 1: Immediate (0-3 months)
1. **GitHub**
   - **Rationale**: GitHub is the primary platform for developers to host and share code. Integrating with GitHub allows users to easily import and export Docker projects.
   - **Free-Tier Limits**: Unlimited public repositories, 500MB storage for private repositories.
   - **Integration Effort**: S
   - **Value-Add**: Solves the job of version control and collaboration for Docker projects.

2. **Docker Hub**
   - **Rationale**: Docker Hub is the default registry for Docker images. Integration allows users to pull and push images directly from the platform.
   - **Free-Tier Limits**: Unlimited public repositories, 1 private repository.
   - **Integration Effort**: S
   - **Value-Add**: Solves the job of image management and distribution.

### Phase 2: Short-Term (3-6 months)
3. **GitLab**
   - **Rationale**: GitLab is a popular alternative to GitHub with built-in CI/CD pipelines. Integration allows users to leverage GitLab's CI/CD features for Docker projects.
   - **Free-Tier Limits**: Unlimited public and private repositories, 400 CI/CD minutes/month.
   - **Integration Effort**: M
   - **Value-Add**: Solves the job of continuous integration and deployment for Docker projects.

4. **AWS ECR**
   - **Rationale**: Amazon Elastic Container Registry (ECR) is a fully-managed Docker container registry. Integration allows users to manage Docker images on AWS.
   - **Free-Tier Limits**: 500MB storage/month for 12 months.
   - **Integration Effort**: M
   - **Value-Add**: Solves the job of secure and scalable image storage.

### Phase 3: Long-Term (6-12 months)
5. **Kubernetes**
   - **Rationale**: Kubernetes is a leading platform for automating deployment, scaling, and management of containerized applications. Integration allows users to deploy Docker projects on Kubernetes clusters.
   - **Free-Tier Limits**: None, but requires a Kubernetes cluster.
   - **Integration Effort**: L
   - **Value-Add**: Solves the job of orchestration and management of containerized applications.

6. **Terraform**
   - **Rationale**: Terraform is an open-source infrastructure as code software tool. Integration allows users to manage infrastructure as code for Docker projects.
   - **Free-Tier Limits**: None, but requires a Terraform Cloud account.
   - **Integration Effort**: L
   - **Value-Add**: Solves the job of infrastructure provisioning and management.

7. **Jenkins**
   - **Rationale**: Jenkins is an open-source automation server. Integration allows users to automate the build, test, and deployment of Docker projects.
   - **Free-Tier Limits**: None, but requires a Jenkins server.
   - **Integration Effort**: L
   - **Value-Add**: Solves the job of automation and continuous delivery.

8. **Travis CI**
   - **Rationale**: Travis CI is a continuous integration service. Integration allows users to automate the testing and deployment of Docker projects.
   - **Free-Tier Limits**: 1 concurrent job, 50 builds/month.
   - **Integration Effort**: M
   - **Value-Add**: Solves the job of continuous integration and deployment.
